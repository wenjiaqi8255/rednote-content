## Project-Specific Patterns

### Test Assertions
- Use `match(/regex/)` over `contains('string')` for HTML with dynamic attributes
- Example: `expect(html).toMatch(/<h1/)` instead of `expect(html).toContain('<h1>')`
- Reason: markdown-it-anchor adds IDs and other attributes to HTML elements

### TypeScript Declarations
- Put custom type declarations in `web/types/` directory
- Don't install `@types/` packages if a simple `.d.ts` file suffices
- Example: `types/markdown-it-katex.d.ts` for PluginSimple type

### CSS Management
- Add new feature styles to `public/assets/card.html` template
- Template is used by `generateXHSCard()` function
- Include KaTeX CSS via CDN link in `<head>`

## Library-Specific Notes

### markdown-it vs marked
- **Why markdown-it?**: Better plugin system, official KaTeX support, better Chinese support
- **Migration impact**: All tests needed updates for HTML attributes added by plugins
- **Configuration**: Requires `transformIgnorePatterns` update in jest.config.js

### Jest + ES Modules
- Must update `transformIgnorePatterns` when installing ES module dependencies
- Pattern: `/node_modules/(?!(package1|package2|...))/`
- Without this, Jest fails to transform the modules

## Common Gotchas

### Build Warnings
- "Detected additional lockfiles" - Can be ignored, relates to parent directory lockfile
- "turbopack.root" warning - Can be ignored, Next.js 16.1.6 message

### Test Failures
- If markdown-it tests fail, check if `transformIgnorePatterns` includes new packages
- If KaTeX tests fail, verify CDN link is in `card.html`
- If header tests fail, update assertions to use `match()` instead of `contains()`
